# 01 — AI Engineering Roadmap

## Big picture

AI engineering is the discipline of building useful software products that use AI models. It is not only research, and it is not only calling an API. A good AI engineer understands enough machine learning to reason about model behavior, enough backend engineering to ship reliable systems, and enough product thinking to solve real user problems.

## AI vs ML vs Deep Learning vs Generative AI

### Artificial Intelligence

Artificial Intelligence is the broad idea of making machines perform tasks that normally require human intelligence: perception, reasoning, planning, language understanding, decision-making, and problem solving.

### Machine Learning

Machine Learning is a subset of AI where systems learn patterns from data instead of being explicitly programmed with every rule.

Example: instead of manually writing every rule for classifying support tickets, you train a model on examples and let it learn patterns.

### Deep Learning

Deep Learning is a subset of machine learning that uses neural networks with many layers. It became especially powerful for images, speech, text, translation, recommendation systems, and modern generative AI.

### Generative AI

Generative AI creates new content: text, images, audio, video, code, summaries, answers, and structured outputs. LLMs are a major type of generative AI focused on language and code.

## What an AI engineer builds

Common products include:

- Chatbots and assistants.
- Document Q&A systems.
- AI search over company knowledge.
- Code generation tools.
- Customer support automation.
- Recommendation systems.
- Content generation tools.
- Data extraction from PDFs, emails, invoices, and forms.
- Agents that use tools and APIs.

## The core skill map

### 1. Programming

You need Python for AI work and usually JavaScript, TypeScript, Dart, Java, or another language for product integration.

### 2. Data thinking

You must understand the quality, format, source, and limitations of data. AI systems fail when the data is weak, biased, messy, outdated, or irrelevant.

### 3. Machine learning basics

You should know features, labels, training data, test data, loss functions, overfitting, generalization, and evaluation metrics.

### 4. LLM application development

You should know prompts, context windows, token limits, model choice, embeddings, retrieval, tool calling, structured output, latency, and cost.

### 5. Backend and deployment

You must ship APIs, manage environment variables, handle errors, secure secrets, log requests, deploy services, and monitor behavior.

### 6. Evaluation

AI output is probabilistic. You need test sets, human review, automated checks, regression tests, and production monitoring.

## Beginner mistake

The common beginner mistake is thinking: AI engineering only means writing prompts.

Better definition: AI engineering means you can design, build, test, and deploy reliable AI-powered applications.

## Practice task

Write a one-page description of an AI product you want to build. Include:

- User problem.
- Input data.
- AI model role.
- Non-AI backend logic.
- Risks.
- How you will evaluate quality.

## Interview questions

2. Why is AI engineering closer to software engineering than pure research?
3. What makes an AI app unreliable?
4. How would you decide whether to use a small or large model?
5. What does evaluation mean in an AI product?
