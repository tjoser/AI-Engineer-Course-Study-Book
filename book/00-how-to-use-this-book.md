# 00 — How to Use This Book

This book is designed for a software engineer who wants to become productive in AI engineering without wasting time passively watching videos.

## What this book is

A practical study companion for an AI engineering course. Each chapter gives you:

- The core ideas you must understand.
- Developer-focused explanations.
- Practical examples.
- Mistakes beginners make.
- Exercises and portfolio tasks.
- Interview-style questions.

## What this book is not

This is not a copied transcript of the videos. Transcripts and course videos are copyrighted, so the book explains the ideas in original wording and adds extra structure for study.

## Study method

For each chapter:

1. Read the concept section.
2. Rewrite the important terms in your own words.
3. Build the practice task.
4. Answer the review questions without looking.
5. Add your own notes.

## Minimum weekly plan

### Week 1: Foundations

- AI vs ML vs Deep Learning vs Generative AI
- Python basics for AI
- Data, features, labels, training, validation

### Week 2: ML and NLP

- Supervised vs unsupervised learning
- Classification, regression, clustering
- Text preprocessing, tokens, embeddings

### Week 3: LLM Applications

- Prompt engineering
- API calls
- Structured outputs
- RAG

### Week 4: Production AI

- LangChain-style app structure
- Vector databases
- Evaluation
- Deployment
- Capstone project

## Best portfolio strategy

Do not only write notes. Build 3 projects:

1. A small LLM API backend.
2. A RAG chatbot over PDFs or documentation.
3. An AI assistant that calls real tools or APIs.

These projects are stronger for jobs than simply saying you watched a course.
